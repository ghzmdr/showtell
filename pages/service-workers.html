Service Workers

SWs are JavaScript scripts that run beneath the main JS application. They're not part of the normal JS flow, they operate closer to the browser.

They can intercept network calls from the main application and manipulate them. This, combined with the possibility of manipulating browser caches, gives SWs an enormous power.
And even better: your main app doesn't have to know anything about this business.

True native applications don't wait for you to open them to display new content, they sync in background. SWs can be 'installed' in the browser so they can fetch updates silently.
Once new content is fetched, say... a new email, maybe you'd like to use push notifications API to let the user know.

These are huge steps towards creating web experiences which are comparable in functionality to native ones, but are more lightweight and flexible.
