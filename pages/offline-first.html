Offline First

The web is slow. Our mobile networks are slow and we feel bad about it.
Native applications don't generate frustrating experiences like a 'You're Offline' page because they're built to be offline first!
The application code resides on the device and doesn't need to be re-downloaded every time. Content can be cached, so the user can enjoy most of the app's functionalities even offline or on bad network conditions.

Building an offline-first web application means to replicate this behavior. Saving the main application in the browser's cache, and fetching content ondemand.

The web platform is supporting this pattern via the new Service Workers API.
